## Key Design Decisions

1. **BackgroundManager.spawn()** used for all 6 sessions (not raw createAgentSession)
2. **Cynefin classification** via regex pattern matching on topic text
3. **Phase 4 judges spawn sequentially** (each awaits completion before next) to handle retry logic
4. **Judge error handling**: retry once on first error, block on second error
5. **Transcripts**: 6 files per debate written to `.autodev/debates/<slug>/`
6. **Commands**: `autodev debate start "topic"` and `autodev debate status` registered via pi.registerCommand()
