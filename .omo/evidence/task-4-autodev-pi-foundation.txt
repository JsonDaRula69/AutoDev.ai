## 5. System Prompt Body Structure

Each agent file follows this structure:
1. YAML frontmatter delimited by `---` fences containing: name, description, tools, model
2. System prompt body = `narrative:` field from source YAML (verbatim)
3. `## Constraints` section with bulleted constraints from source YAML
4. `## Capabilities` section with bulleted capabilities from source YAML

For the 4 operations agents (quartermaster, boatswain, navigator, watch-officer),
the system prompt body comes from engineer.yaml's narrative + constraints +
capabilities, while the frontmatter (name, description, tools, model) comes
from each agent's own YAML file.

For explore.md, the narrative, constraints, and capabilities were authored
from scratch in Nautilus submarine theme.
