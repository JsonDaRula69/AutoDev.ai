## Architecture

- `loadGuardrailsConfig(projectRoot)` parses `.autodev/config/guardrails.yaml`
  using a focused YAML reader (no external yaml dep). Returns hard_stops + soft_stops.
- `buildHandler(config, deps?)` returns an async `(event, projectRoot) => RuleResult`
  function. Tests import this directly with a temp project root and a
  mock ciChecker — no pi runtime needed.
- `register(pi)` wires the handler to `pi.on("tool_call", ...)`, mapping
  `{ block, reason }` to `{ block: true, reason }` and emitting soft-stop
  warnings via `ctx.ui.notify(msg, "warning")`.

## Hard Stops Implemented (block)

1. never-deploy-directly — blocks bash with deploy-like commands (deploy,
   kubectl/terraform/helm apply).
2. no-secrets-in-code — blocks write/edit containing secret patterns
   (sk-ant-, sk-or-, AIza, ghp_, github_pat_, xox[baprs]-, PEM private key,
   eyJ JWT header).
3. evidence-or-it-didnt-happen — blocks `git commit` when `.omo/evidence/`
   has no .md/.txt files. (Uses .omo/evidence/ per ARCHITECTURE.md §6, not
   the stale .autodev/evidence/ in AGENTS.md.)
4. one-task-at-a-time — blocks todowrite in_progress when a different task
   is active in `.autodev/active-task.json`. Records new active tasks;
   clears on completion.
5. follow-the-plan — blocks write/edit to paths not mentioned in any
   `.omo/plans/*.md`. No-plan → allow (nothing to deviate from).
6. ci-is-the-hard-gate — blocks `gh pr merge` when the injected ciChecker
   returns false. Default checker shells out to `gh pr checks --required`.
7. never-approve-own-work — blocks review tool calls where
   reviewer == implementer.
8. never-modify-reference-docs — blocks write/edit to .autodev/reference/.
9. never-modify-debate-transcripts — blocks write/edit to .autodev/debates/.

## Soft Stops Implemented (warn — never block)

- warn-scope — edit with >10 edits.
- flag-missing-evidence — review with no evidence file.
- suggest-review — write larger than 2000 chars.
